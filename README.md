# Visitor
Посетитель — это поведенческий паттерн проектирования, который позволяет добавлять в программу новые операции, не изменяя классы объектов, над которыми эти операции могут выполняться
(из книги "Погружение в паттерны проектирования")

Обладает следующими характеристиками:
Тип:  поведенческий

Принцип работы:

1. Создается интерфейс посетителя, который определяет методы, которые должны быть реализованы всеми посетителями.

2. Создаются конкретные классы посетителей, которые реализуют методы посетителя и выполняют различные операции над элементами структуры.

3. Создается структура объектов, которая содержит элементы, которые можно посетить.

4. Структура объектов принимает посетителя и вызывает соответствующие методы посетителя для каждого элемента, который нужно посетить.

5. Посетитель может выполнять различные операции над элементами, например, собирать информацию, изменять состояние или выполнять другие действия.


Назначение: Применение одной операции над объектами разных классов или сложной структуры.
Метафора: Представьте начинающего страхового агента, жаждущего получить новых клиентов. Он беспорядочно посещает все дома в округе, предлагая свои услуги. Но для каждого из посещаемых типов домов у него имеется особое предложение. 

 • Придя в дом к обычной семье, он предлагает оформить медицинскую страховку.

 • Придя в банк, он предлагает страховку от грабежа.

 • Придя на фабрику, он предлагает страховку предприятия от пожара и наводнения.


Потенциальные проблемы: Можно нарушить инкапсуляцию, сложно и бессмысленно реализовывать, если  иерархия элементов часто меняется.
